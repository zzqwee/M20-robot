# M20 机器狗控制器 - 开发过程文档

## 概述

本文档记录了 M20 机器狗桌面应用控制器的完整开发过程，包括需求分析、技术选型、项目初始化、功能实现等各个阶段。

## 阶段一：需求确认

### 用户需求来源

用户提供了初步需求文档 `初步需求.md`，包含以下要求：

1. **设备信息获取** - 设备型号、设备版本、设备序列号、设备名称、设备描述、设备状态、设备IP地址、设备MAC地址、设备网关地址、设备子网掩码、设备DNS地址、设备时钟

2. **运动控制** - 移动（前进、后退、左移、右移）、旋转（左转、右转）、停止

3. **LED灯控制** - 点亮、熄灭、闪烁

4. **麦克风控制** - 开启、关闭、调整音量

5. **摄像头控制** - 开启、关闭、调整分辨率

6. **传感器读取** - 距离、角度、温度

7. **电池信息** - 电压、状态

### UI 设计要求

用户指定了 Apple FrostyGlass 设计风格：
- 毛玻璃效果（frosty glass）
- 浅色背景 (#F5F7FA)
- 现代化、极简设计
- 参考 Apple Support 网站和 Qwen AI 界面

### 核心功能优先级

1. IP配置和连接 - 最高优先级
2. 设备信息获取 - 最高优先级
3. 运动控制 - 最高优先级
4. LED控制 - 中优先级
5. 传感器/电池读取 - 中优先级
6. 音视频控制 - 低优先级

### 技术选型

根据用户建议和现代桌面应用开发最佳实践：
- **框架**: Electron（跨平台桌面应用）
- **前端**: React 18 + TypeScript
- **状态管理**: Zustand
- **通信**: 原生 WebSocket/Socket TCP
- **构建工具**: Vite
- **打包工具**: electron-builder

## 阶段三：Electron 后端开发

### 创建主进程文件

`electron/main.ts` - 主进程入口，处理窗口管理和 Socket 通信：

```typescript
import { app, BrowserWindow, ipcMain } from 'electron'
import { join } from 'path'
import * as net from 'net'

let mainWindow: BrowserWindow | null = null
let socket: net.Socket | null = null
let isConnected = false
const DEFAULT_TCP_PORT = 30001
const DEFAULT_UDP_PORT = 30000

function createWindow() {
  mainWindow = new BrowserWindow({
    width: 1200,
    height: 800,
    minWidth: 900,
    minHeight: 600,
    webPreferences: {
      preload: join(__dirname, 'preload.js'),
      nodeIntegration: false,
      contextIsolation: true
    },
    frame: true,
    backgroundColor: '#F5F7FA'
  })

  if (process.env.NODE_ENV === 'development' || process.env.VITE_DEV_SERVER_URL) {
    mainWindow.loadURL(process.env.VITE_DEV_SERVER_URL || 'http://localhost:5173')
    mainWindow.webContents.openDevTools()
  } else {
    mainWindow.loadFile(join(__dirname, '../dist/index.html'))
  }
}

// Socket 连接处理函数
function connectToDevice(host: string, port: number): Promise<void> {
  return new Promise((resolve, reject) => {
    // 创建 TCP Socket 连接
    socket = new net.Socket()
    socket.connect(port, host, () => {
      isConnected = true
      resolve()
    })
    socket.on('error', (err) => {
      isConnected = false
      reject(err)
    })
    socket.on('close', () => {
      isConnected = false
      if (mainWindow) {
        mainWindow.webContents.send('connection-closed')
      }
    })
    socket.on('data', (data) => {
      if (mainWindow) {
        try {
          const message = JSON.parse(data.toString())
          mainWindow.webContents.send('socket-message', message)
        } catch (e) {
          console.error('Failed to parse message:', e)
        }
      }
    })
  })
}

// IPC 处理器
ipcMain.handle('connect', async (event, host: string, port: number) => {
  try {
    await connectToDevice(host, port || DEFAULT_PORT)
    return { success: true }
  } catch (error: any) {
    return { success: false, error: error.message }
  }
})

ipcMain.handle('disconnect', () => {
  disconnect()
  return { success: true }
})

ipcMain.handle('send-command', async (event, command: object) => {
  try {
    await sendCommand(command)
    return { success: true }
  } catch (error: any) {
    return { success: false, error: error.message }
  }
})

ipcMain.handle('get-connection-status', () => {
  return { connected: isConnected }
})
```

### 创建预加载脚本

`electron/preload.ts` - 暴露安全的 IPC 接口给渲染进程：

```typescript
import { contextBridge, ipcRenderer } from 'electron'

export interface ElectronAPI {
  connect: (host: string, port: number) => Promise<{ success: boolean; error?: string }>
  disconnect: () => Promise<{ success: boolean }>
  sendCommand: (command: object) => Promise<{ success: boolean; error?: string }>
  getConnectionStatus: () => Promise<{ connected: boolean }>
  onSocketMessage: (callback: (message: any) => void) => void
  onConnectionClosed: (callback: () => void) => void
}

const electronAPI: ElectronAPI = {
  connect: (host: string, port: number) => ipcRenderer.invoke('connect', host, port),
  disconnect: () => ipcRenderer.invoke('disconnect'),
  sendCommand: (command: object) => ipcRenderer.invoke('send-command', command),
  getConnectionStatus: () => ipcRenderer.invoke('get-connection-status'),
  onSocketMessage: (callback: (message: any) => void) => {
    ipcRenderer.on('socket-message', (event, message) => callback(message))
  },
  onConnectionClosed: (callback: () => void) => {
    ipcRenderer.on('connection-closed', () => callback())
  }
}

contextBridge.exposeInMainWorld('electronAPI', electronAPI)
```

### 创建状态管理

`src/stores/useAppStore.ts` - 使用 Zustand 管理应用状态：

```typescript
import { create } from 'zustand'
import { DeviceInfo, BatteryInfo, SensorData, ConnectionStatus } from '../types'

interface AppState {
  connection: ConnectionStatus
  deviceInfo: DeviceInfo | null
  batteryInfo: BatteryInfo | null
  sensorData: SensorData | null

  setConnection: (status: Partial<ConnectionStatus>) => void
  setDeviceInfo: (info: DeviceInfo | null) => void
  setBatteryInfo: (info: BatteryInfo | null) => void
  setSensorData: (data: SensorData | null) => void
}

export const useAppStore = create<AppState>((set) => ({
  connection: { connected: false, host: '', port: 30001 },
  deviceInfo: null,
  batteryInfo: null,
  sensorData: null,

  setConnection: (status) =>
    set((state) => ({ connection: { ...state.connection, ...status } })),
  setDeviceInfo: (info) => set({ deviceInfo: info }),
  setBatteryInfo: (info) => set({ batteryInfo: info }),
  setSensorData: (data) => set({ sensorData: data })
}))
```

### 创建主应用组件

`src/App.tsx` - 应用主入口，整合所有组件：

```tsx
function App() {
  const { connection, setConnection, setDeviceInfo, setBatteryInfo, setSensorData } = useAppStore()

  useEffect(() => {
    if (window.electronAPI) {
      window.electronAPI.onSocketMessage((message) => {
        handleMessage(message)
      })
      window.electronAPI.onConnectionClosed(() => {
        setConnection({ connected: false, error: 'Connection closed' })
      })
    }
  }, [])

  const handleMessage = (message: any) => {
    if (message.ret === 0) {
      const data = message.data
      if (data.type === 'device_info') {
        setDeviceInfo(data.info)
      } else if (data.type === 'battery') {
        setBatteryInfo(data.info)
      } else if (data.type === 'sensor') {
        setSensorData(data.info)
      }
    }
  }

  return (
    <div className="app-container">
      <Navbar />
      <main className="main-content">
        <div className="hero-card glass-heavy">
          <h1>M20 机器狗控制器</h1>
          <ConnectionPanel />
        </div>
        {connection.connected && (
          <div className="cards-grid">
            <DeviceInfoCard />
            <BatteryPanel />
            <SensorPanel />
          </div>
        )}
        {connection.connected && (
          <div className="control-panel">
            <MovementControl />
            <LEDControl />
            <MicrophoneControl />
            <CameraControl />
          </div>
        )}
      </main>
    </div>
  )
}
```

## 阶段五：依赖安装与问题解决

### 第一次安装依赖

```bash
npm install
```

**问题**：安装过程中卡住，electron 包下载超时。

**解决**：手动设置 npm 镜像并重新安装 electron：

```bash
# 设置 npm 镜像
npm config set registry https://registry.npmmirror.com/
export ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/"

# 删除损坏的 electron
rm -rf node_modules/electron

# 重新安装 electron
ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/" npm install electron@28.0.0 --save-dev
```

### Electron 安装错误

**问题**：Electron 报错 "Electron failed to install correctly"

**原因**：electron 包文件损坏或不完整

**解决**：删除 node_modules/electron 目录，使用正确的镜像重新安装
