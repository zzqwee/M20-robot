# M20机器狗控制应用开发过程

**项目名称**: M20机器狗控制应用
**开发日期**: 2026-03-18
**版本**: V1.0

---

## 一、需求确认

### 1.1 初步需求分析

根据用户提供的初步需求文档，需要开发一款连接M20/M20 Pro机器狗的Web应用，主要功能包括：

1. **设备信息获取** - 设备型号、版本、序列号、状态、IP地址、MAC地址等
2. **运动控制** - 移动、旋转、停止
3. **LED控制** - 闪烁、熄灭、点亮
4. **麦克风控制** - 开启、关闭、音量调整
5. **摄像头控制** - 开启、关闭、分辨率调整
6. **传感器读取** - 距离、角度、温度
7. **电池状态** - 电压、状态

### 1.2 机器狗通信协议分析

根据M20软件开发指南文档，分析得到以下通信协议：

- **UDP端口**: 30000 (接收状态上报)
- **TCP端口**: 30001 (发送控制指令)
- **协议格式**: 16字节头部 + JSON ASDU
- **默认IP**: 192.168.10.104 (用户更新)

### 1.3 UI设计要求

采用Apple FrostyGlass设计风格：
- 毛玻璃效果导航栏
- 淡蓝色/灰色渐变背景 (#F5F7FA)
- 卡片式布局
- 主色调: #007AFF (Apple蓝)

## 二、计划编制

### 2.1 技术架构

采用前后端分离架构：

| 层级 | 技术选型 |
|------|----------|
| 前端框架 | React 18 + TypeScript |
| 构建工具 | Vite |
| 样式方案 | TailwindCSS |
| 后端运行时 | Node.js + TypeScript |
| 通信协议 | WebSocket (前后端), TCP/UDP (后端-机器狗) |
| 应用端口 | 5173 (前端), 3000 (后端WebSocket) |

### 2.2 功能模块划分

| 模块 | 优先级 | 说明 |
|------|--------|------|
| 设备连接 | P1 | 配置IP连接机器狗 |
| 设备信息 | P1 | 显示设备详细信息 |
| 运动控制 | P1 | 方向按钮控制移动 |
| LED控制 | P2 | 开关和闪烁模式 |
| 电池监控 | P2 | 电压、电量显示 |
| 模拟数据 | P2 | 不连接设备时测试UI |

### 3.2 核心功能实现

#### 后端实现

1. **协议解析** (`protocol/header.ts`, `protocol/messages.ts`)
   - 16字节协议头部解析
   - JSON格式消息构建
   - 心跳、运动控制、LED控制等消息类型

2. **TCP客户端** (`tcp-client.ts`)
   - 连接机器狗TCP端口30001
   - 发送控制指令
   - 处理连接状态

3. **UDP服务器** (`udp-server.ts`)
   - 监听UDP端口30000
   - 接收设备状态上报
   - 解析设备信息、电池、传感器数据

4. **WebSocket服务器** (`index.ts`)
   - 前端WebSocket连接处理
   - 消息路由（连接、运动、LED、设备控制）
   - 模拟数据模式支持

#### 前端实现

1. **API服务** (`services/api.ts`)
   - WebSocket连接管理
   - 请求/响应处理
   - 状态更新回调

2. **状态管理** (`stores/robotStore.ts`)
   - 设备连接状态
   - 设备信息、电池、传感器数据
   - 模拟模式开关

3. **UI组件**
   - `ConnectionPanel`: IP配置和连接控制
   - `DeviceInfo`: 设备信息显示
   - `MotionControl`: 方向按钮控制面板
   - `LEDControl`: LED开关和闪烁
   - `BatteryStatus`: 电池电量显示
   - `SensorData`: 传感器数据展示
   - `DeviceToggle`: 摄像头、麦克风开关

### 3.3 模拟数据模式

在设置页面提供"模拟模式"开关，开启后：
- 后端自动生成模拟的设备信息
- 每2秒推送模拟数据
- 可测试所有UI功能，无需连接真实机器狗
