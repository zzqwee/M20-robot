# Feature Specification: M20机器狗控制应用

**Feature Branch**: `001-m20-robot-control`
**Created**: 2026-03-18
**Status**: Draft
**Input**: User description: "创建M20机器狗控制应用的需求开发规范文档，包括：1. 获取设备信息（型号、版本、序列号、状态、IP地址、MAC地址等）2. 控制运动（移动、旋转、停止）3. 控制LED灯4. 控制麦克风5. 控制摄像头6. 读取传感器数据7. 读取电池状态。UI采用Apple FrostyGlass风格，毛玻璃效果，浅色背景。"

## 1. Overview

本规范文档描述一款连接M20/M20 Pro机器狗的Web应用系统，用于获取设备详细信息并实现对机器狗的基本控制功能。

**应用端口**: 5173

## 2. User Scenarios & Testing

### User Story 1 - 设备连接与信息查看 (Priority: P1)

用户通过手动配置机器狗IP地址，成功连接到设备后，能够查看到机器狗的全部设备信息。

**Why this priority**: 设备连接和信息查看是本应用的核心功能，是用户使用本应用的首要场景。

**Independent Test**: 用户配置IP地址后点击连接，系统显示连接状态，成功后展示设备详细信息卡片。

**Acceptance Scenarios**:

1. **Given** 用户打开应用，**When** 用户输入有效IP地址并点击连接按钮，**Then** 系统显示"连接中"状态，随后显示"连接成功"并展示设备信息
2. **Given** 用户输入无效IP地址，**When** 用户点击连接按钮，**Then** 系统显示"连接失败"并提示错误信息
3. **Given** 应用已连接设备，**When** 用户断开连接或设备离线，**Then** 系统显示"离线"状态并更新UI

---

### User Story 2 - 运动控制 (Priority: P1)

用户通过方向控制面板，能够控制机器狗前进、后退、左移、右移、停止以及左转右转。

**Why this priority**: 运动控制是机器狗应用的核心交互功能，用户需要能够直观地控制机器狗移动。

**Independent Test**: 用户在运动控制面板点击方向按钮，机器狗按预期方向移动。

**Acceptance Scenarios**:

1. **Given** 设备已连接，**When** 用户点击"前进"按钮，**Then** 机器狗向前移动
2. **Given** 设备已连接，**When** 用户点击"停止"按钮，**Then** 机器狗立即停止移动
3. **Given** 设备已连接，**When** 用户点击"左转"按钮，**Then** 机器狗原地向左旋转

---

### User Story 3 - LED灯光控制 (Priority: P2)

用户能够控制机器狗LED灯的开关和闪烁模式。

**Why this priority**: LED控制是机器狗的视觉反馈功能，属于常用的扩展功能。

**Independent Test**: 用户点击LED控制按钮，机器狗灯光按预期变化。

**Acceptance Scenarios**:

1. **Given** 设备已连接，**When** 用户点击"开灯"按钮，**Then** 机器狗LED灯点亮
2. **Given** 设备已连接，**When** 用户点击"关灯"按钮，**Then** 机器狗LED灯熄灭
3. **Given** 设备已连接，**When** 用户点击"闪烁"按钮，**Then** 机器狗LED灯按一定频率闪烁

---

### User Story 4 - 电池状态监控 (Priority: P2)

用户能够实时查看机器狗电池的电压、电量和充放电状态。

**Why this priority**: 电池状态是用户关注机器狗工作状态的重要指标，影响使用时间规划。

**Independent Test**: 用户在电池状态面板查看电池信息，数据与实际一致。

**Acceptance Scenarios**:

1. **Given** 设备已连接，**When** 电池正常工作，**Then** 显示左右电池电压、电量百分比
2. **Given** 设备已连接，**When** 电池电量低于20%，**Then** 显示低电量警告

---

### User Story 5 - 传感器数据读取 (Priority: P3)

用户能够查看机器狗传感器的实时数据，包括距离、角度、温度等信息。

**Why this priority**: 传感器数据用于了解机器狗周围环境和自身状态，属于辅助功能。

**Independent Test**: 用户在传感器面板查看实时数据，数据定期更新。

**Acceptance Scenarios**:

1. **Given** 设备已连接，**When** 传感器正常工作，**Then** 显示激光雷达测距数据
2. **Given** 设备已连接，**When** IMU正常工作，**Then** 显示姿态角度数据

---

### User Story 6 - 模拟数据模式 (Priority: P2)

在开发测试阶段，用户无需连接真实机器狗即可预览所有页面功能和UI效果。

**Why this priority**: 方便开发人员测试UI和功能流程，无需每次都连接真实设备。

**Independent Test**: 用户开启模拟数据模式后，所有页面显示模拟数据，关闭后恢复正常。

**Acceptance Scenarios**:

1. **Given** 用户未连接设备，**When** 用户开启"模拟数据"开关，**Then** 所有页面显示模拟的设备信息和控制界面
2. **Given** 用户在模拟数据模式，**When** 用户点击运动控制按钮，**Then** UI显示控制反馈（无需真实发送指令）
3. **Given** 用户关闭模拟数据模式，**Then** 页面恢复需连接设备状态

---

### User Story 7 - 摄像头和麦克风控制 (Priority: P3)

用户能够控制机器狗摄像头和麦克风的开关状态。

**Why this priority**: 摄像头和麦克风是机器狗的音视频采集设备，属于扩展功能。

**Independent Test**: 用户点击控制按钮，系统显示设备开关状态。

**Acceptance Scenarios**:

1. **Given** 设备已连接，**When** 用户点击"开启摄像头"按钮，**Then** 摄像头状态显示为开启
2. **Given** 设备已连接，**When** 用户点击"关闭麦克风"按钮，**Then** 麦克风状态显示为关闭

---

### Edge Cases

- 网络断开时，应用应自动显示离线状态并提示用户
- 设备响应超时时，应显示相应错误提示
- 运动控制时发送频率建议为20Hz，避免指令丢失

## 3. Requirements

### Functional Requirements

- **FR-001**: 系统必须支持用户手动输入机器狗IP地址进行连接
- **FR-002**: 系统必须通过TCP协议（端口30001）发送控制指令
- **FR-003**: 系统必须通过UDP协议（端口30000）接收设备状态上报
- **FR-004**: 系统必须能获取并显示设备型号（Version）、设备序列号信息
- **FR-005**: 系统必须能获取并显示设备状态（运行/休眠/充电等）
- **FR-006**: 系统必须能获取并显示设备IP地址、MAC地址信息
- **FR-007**: 系统必须支持运动控制指令（前进、后退、左移、右移、左转、右转、停止）
- **FR-008**: 运动控制指令发送频率应支持20Hz
- **FR-009**: 系统必须能控制LED灯的开启、关闭、闪烁模式
- **FR-010**: 系统必须能读取电池电压、电量百分比、充放电状态
- **FR-011**: 系统必须能读取激光雷达距离数据
- **FR-012**: 系统必须能读取IMU姿态角度数据
- **FR-013**: 系统必须能控制摄像头的开启和关闭
- **FR-014**: 系统必须能控制麦克风的开启和关闭
- **FR-015**: UI必须采用Apple FrostyGlass设计风格（毛玻璃效果、浅色背景）
- **FR-016**: 连接状态必须实时显示（已连接、连接中、离线、连接失败）
- **FR-017**: 系统必须支持模拟数据模式，在未连接真实机器狗时可查看所有页面功能
- **FR-018**: 系统必须提供模拟数据开关，方便开发和测试

### Key Entities

- **设备连接**: 包含IP地址、端口号、连接状态
- **设备信息**: 型号、版本、序列号、状态、IP地址、MAC地址
- **运动指令**: X轴速度、Y轴速度、Yaw角速度
- **电池状态**: 电压、电量百分比、充放电状态
- **传感器数据**: 激光雷达距离、IMU角度、温度

## 4. Success Criteria

### Measurable Outcomes

- **SC-001**: 用户能在30秒内完成从打开应用到成功连接设备的全流程
- **SC-002**: 运动控制指令延迟不超过100毫秒
- **SC-003**: 设备状态信息每2秒自动更新一次
- **SC-004**: 100%的有效IP地址连接请求能在10秒内得到响应
- **SC-005**: 用户满意度评分达到4分以上（5分制）

## 5. Assumptions

- 假设机器狗的默认IP地址为192.168.10.104，但用户可手动配置
- 假设用户具备基本的网络知识，能够配置同一局域网内的IP地址
- 假设机器狗已开启TCP/UDP通信服务
- 假设应用主要在PC浏览器中使用
- 假设电池为双电池配置（左侧和右侧）
- 假设开发测试阶段需要模拟数据模式以便不连接真实机器狗进行UI测试
