# Research: M20机器狗控制应用技术研究

**Date**: 2026-03-18
**Feature**: M20机器狗控制应用

## 技术选型

### 后端语言选型

| 选项 | 优点 | 缺点 | 决策 |
|------|------|------|------|
| Node.js | 前后端TypeScript统一，TCP/UDP支持完善 | 单线程 | **采用** |
| Python | 协议处理方便 | 与前端技术栈不统一 | 备用 |
| Go | 高并发，性能好 | 学习成本 | 备用 |

### 前端框架选型

| 选项 | 优点 | 决策 |
|------|------|------|
| React 18 | 社区成熟，组件化 | **采用** |
| Vue 3 | 上手简单 | 备用 |
| Vanilla JS | 轻量 | 不采用 |

### 构建工具

| 选项 | 优点 | 决策 |
|------|------|------|
| Vite | 快速启动，热更新 | **采用** |
| Webpack | 成熟稳定 | 备用 |

## M20协议研究

### 通信架构

```
┌─────────────┐    WebSocket     ┌─────────────┐    TCP     ┌─────────────┐
│   前端React  │◄────────────────►│  后端Node   │◄──────────►│  机器狗M20  │
│   端口5173   │                  │  通信服务    │   端口30001 │             │
└─────────────┘                  └─────────────┘            └─────────────┘
                                       ▲
                                       │ UDP
                                       │ 端口30000
                                       ▼
                                ┌─────────────┐
                                │  状态上报   │
                                └─────────────┘
```

### 协议头部结构 (16字节)

| 字段 | 长度 | 说明 |
|------|------|------|
| HeaderFlag | 2字节 | 帧头标识 (0xEB90) |
| DataLength | 2字节 | 数据长度 |
| ProtocolType | 1字节 | 协议类型 (0=JSON, 1=XML) |
| MessageType | 2字节 | 消息类型 |
| CommandCode | 2字节 | 命令码 |
| SequenceNum | 2字节 | 序列号 |
| Reserved | 3字节 | 保留字段 |
| Data | 变长 | ASDU数据 |

### 关键消息类型

| 消息类型 | Type | Command | 说明 |
|----------|------|---------|------|
| 心跳指令 | 100 | 100 | 维持连接，触发状态上报 |
| 运动控制 | 2 | 21 | 轴指令控制 |
| 步态切换 | 2 | 23 | 切换运动步态 |
| LED控制 | 1101 | 2 | 照明灯控制 |
| 设备状态 | 1002 | 5 | 2Hz上报 |
| 运控状态 | 1002 | 4 | 10Hz上报 |
| 电池状态 | 1002 | 5 | 包含在设备状态中 |

## UI设计研究

### Apple FrostyGlass风格

根据需求文档，UI设计要点：

1. **毛玻璃效果**: 使用CSS backdrop-filter: blur()
2. **浅色背景**: #F5F7FA
3. **卡片式布局**: 圆角卡片，带轻微阴影
4. **导航栏**: 半透明毛玻璃效果
5. **配色**: Apple蓝 #007AFF 作为主色

### 页面结构

1. **Dashboard页面**: 显示设备信息、连接状态、电池状态
2. **Control页面**: 运动控制、LED控制、设备开关
3. **Settings页面**: IP配置、模拟数据开关

## 模拟数据模式

### 设计目标

- 在不连接真实机器狗的情况下测试UI和功能
- 模拟所有设备数据和状态变化
- 方便开发和演示

### 模拟数据内容

- 设备信息: 型号M20 Pro，序列号SIMU001
- 连接状态: 模拟连接成功
- 电池: 模拟电压24V，电量80%
- 运动控制: UI反馈但不发送真实指令
